If you have tried to use Paypal's API to create payments for your website, [you know how painful the experience can be](https://jonathanprior.com/2011/07/11/paypals-dreadful-api-and-documentation/).
For some reason, there is no code snippet on how to implement [adaptive payments](https://developer.paypal.com/docs/classic/adaptive-payments/integration-guide/APIntro/) (parallel and chain) in Python.
 
Note that for some reason their REST API does not support this particular type of payment, so you can't use
[paypalrestsdk](https://github.com/paypal/PayPal-Python-SDK).

from the REST API; this is something I found out only by emailing their support, it's not written in their (otherwise incredibly
verbose/bloated/redundant) documentation. You are stuck with the older [NVP API](https://developer.paypal.com/docs/classic/api/NVPAPIOverview/) for this.

Below some simple code snippets that will help you get started.

First we create the necessary `dicts`

{% highlight python %}
import requests
import json

payload_check_verify = {
    'emailAddress': paypal_email,
    # accountIdentifier
    'firstName': firstname,
    'lastName': lastname,
    'matchCriteria': 'NAME',  # NONE, NAME
    'requestEnvelope': {
        'errorLanguage': 'en_US',
        'detailLevel': 'ReturnAll'
    }
}

r = requests.post(PAYPAL_API_ENDPOINT_CHECK_VERIFIED, data=json.dumps(payload_check_verify), headers=PAYPAL_HEADERS)
if r.json()['responseEnvelope']['ack'] == 'Success':
    return True
else:
    return False
{% endhighlight %}
